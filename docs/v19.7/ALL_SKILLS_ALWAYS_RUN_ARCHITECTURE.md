## Skills Execution Flow - Always-On

### When Processing a Prompt

```
1. PTD (Prompt Template Design)
   └─ Decompose: "Build a React app" → [Task 1, Task 2, Task 3]

2. SR (Semantic Routing) + GAR (Gradient-Aware Routing)
   └─ Route: [Task 1 → DiveCoder-1, Task 2 → DiveCoder-2, Task 3 → DiveCoder-3]

3. DCA (Dynamic Capacity Allocation)
   └─ Allocate: CPU, Memory, GPU based on task complexity

4. HE (Hierarchical Experts)
   └─ Decompose: Complex tasks into sub-tasks for specialists

5. CAC (Context-Aware Compression)
   └─ Compress: Large contexts to essential information

6. TA (Temporal Attention)
   └─ Focus: On recent and relevant information

7. CPCG (Contextual Prompt Completion Generation)
   └─ Generate: Appropriate prompts for each task

8. SCW (Semantic Context Weighting)
   └─ Weight: Semantic importance of context

9. SHC (Semantic Hierarchical Clustering)
   └─ Cluster: Related information together

10. CCF (Cross-Context Fusion)
    └─ Fuse: Multiple contexts into unified view

11. DRC (Dynamic Reasoning Chain)
    └─ Reason: Through problem dynamically

12. DNAS (Dynamic Neural Architecture Search)
    └─ Search: For optimal model architecture

13. HDS (Hybrid Dense-Sparse)
    └─ Compute: Using efficient sparse layers

14. ITS (Inference-Time Scaling)
    └─ Scale: Resources based on task priority

15. MVP (Multi-View Processing)
    └─ Verify: From multiple perspectives

16. FPV (Formal Program Verification)
    └─ Verify: Mathematical correctness

17. AEH (Automatic Error Handling)
    └─ Handle: Any errors with recovery

18. EGFV (Error Gradient Flow Visualization)
    └─ Visualize: Error flows for debugging

19. UFBL (User Feedback-Based Learning)
    └─ Learn: From user feedback

20. CLLT (Continuous Learning with Long-Term Memory)
    └─ Remember: Solutions for future use

21. FEL (Federated Expert Learning)
    └─ Share: Learning across instances

22. CEKS (Cross-Expert Knowledge Sharing)
    └─ Distribute: Knowledge to all experts

23. Output: Final verified, optimized code
```
